# Phase 4 - Site Pages & Content

## Objective
Finalize information architecture and complete production-ready content for all core, SEO, and trust pages.

## Success Criteria
- Final sitemap approved.
- Every required page has final copy, media, CTAs, metadata, and internal links.
- Health/wellness trust requirements are embedded (credentials, disclaimers, evidence-aware language).
- Content is ready for direct implementation in the frontend.

## Inputs Required
- Discovery assets from Phase 1.
- Design system and mockups from Phase 3.
- Keyword tiers and SEO strategy from research plan.

## Step-by-Step Tasks

### 1. Final Sitemap and URL Structure
1. Lock final URL structure (short, readable, keyword-aligned slugs).
2. Define canonical page list and parent-child hierarchy.
3. Create redirect map from old Square URLs to new URLs.
4. Reserve URL namespaces for future growth (blog categories, workshops/retreats).

### 2. Content Model Definition
Define required fields per content type.

Content types:
- Standard page.
- Product page.
- Service/coaching page.
- FAQ item.
- Blog post.
- Testimonial.
- Policy page.

For each type, define:
- Required headings.
- Body content sections.
- CTA type and destination.
- Required media.
- SEO fields (title/meta/OG/canonical).
- Schema fields.

### 3. Page-by-Page Content Production
For each page, write final content and map required elements:

Core pages:
- Homepage.
- Shop.
- The Sacred Portal Path.
- About the Creator.
- Testimonials.
- Workshops & Retreats (Launching Soon).
- Contact.

SEO/trust pages:
- What is Yoni Steaming?
- FAQ.
- San Diego Wellness Services.
- Blog index.
- Disclaimer.
- Return & Exchange Policy.

For every page, include:
- H1.
- Supporting H2/H3 structure.
- Primary CTA and secondary CTA.
- Evidence/trust block (credentials, social proof, policy clarity).
- Internal links to priority pages.

### 4. Product and Coaching Content Normalization
1. Normalize product descriptions to a consistent structure:
- Summary.
- Benefits.
- Ingredients/materials.
- Usage guidance.
- Safety/disclaimer language.
- Shipping/returns note.
2. Normalize coaching offer content:
- Who it is for.
- Problems addressed.
- Program structure.
- Outcomes.
- Enrollment CTA.
- Case-by-case return policy note.

### 5. Tone and Compliance Pass
1. Run tone review for consistent voice across all pages.
2. Remove unsupported medical claims and absolute language.
3. Add disclaimers where needed.
4. Ensure RN/BSN/HC-NC credentials are represented accurately and consistently.

### 6. Metadata and Social Previews
For every indexable page, create:
- SEO title.
- Meta description.
- OG title/description/image.
- Canonical URL.
- Suggested social share text.

Create a metadata matrix in `content-metadata.csv`.

### 7. Internal Linking Architecture
1. Map pillar pages and support pages.
2. Add contextual links between:
- Coaching page and testimonials.
- Product pages and educational pages.
- Blog posts and service pages.
3. Ensure no orphan pages remain.

### 8. Editorial Workflow and Approvals
1. Create copy doc set for review comments.
2. Define revision rounds and lock dates.
3. Track page status:
- Draft.
- In review.
- Approved.
- Ready for implementation.
4. Publish final approved copy package into `/planning/content/final`.

## Deliverables
- Approved sitemap and URL map.
- Completed copy for all required pages.
- Metadata matrix.
- Internal linking map.
- Final content package ready for dev integration.

## Quality Control Checklist
- Every page has defined CTA(s).
- Every page has metadata.
- Every page has required trust and policy signals.
- No duplicate/conflicting claims.
- No missing legal/policy pages.

## Risks and Mitigations
- Content delays: prioritize revenue-critical pages first (coaching + top products).
- Overly long copy: keep scannable structure and progressive disclosure.
- SEO keyword stuffing risk: keep language natural and reader-first.
